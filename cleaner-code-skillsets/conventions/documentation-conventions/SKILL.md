# Documentation Conventions

Discover local style before recommending changes.

## Reference

Use `references/convention-discovery.md`.

## Check

- Existing README, comment, docstring, and example style.
- Where the project documents setup, usage, decisions, and operations.
- Level of detail expected for public versus internal APIs.

## Recommend

- Update docs only where they help future work.
- Keep examples executable or clearly scoped.
- Remove stale comments instead of expanding them.
