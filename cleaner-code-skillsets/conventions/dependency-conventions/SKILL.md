# Dependency Conventions

Discover local style before recommending changes.

## Reference

Use `references/convention-discovery.md`.

## Check

- Existing dependency files, import style, and version policy.
- Preferred standard-library or local helper alternatives.
- Runtime, development, and test dependency boundaries.

## Recommend

- Avoid new packages unless value exceeds maintenance cost.
- Keep dependency scope and import style local.
- Note security, license, or update concerns.
