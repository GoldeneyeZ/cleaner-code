# Testing Conventions

Discover local style before recommending changes.

## Reference

Use `references/convention-discovery.md`.

## Check

- Nearby test naming, setup, assertion, and fixture patterns.
- Preferred test runner commands and helper APIs.
- Local policy on mocks, integration tests, and characterization tests.

## Recommend

- Match existing test shape unless it weakens behavior coverage.
- Keep fixtures explicit enough to show intent.
- Record the focused verification command.
