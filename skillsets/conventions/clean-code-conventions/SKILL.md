# Clean Code Conventions

Discover local style before recommending changes.

## Reference

Use `skillsets/shared/references/convention-discovery.md`.

## Check

- Nearby function size, naming, comments, and control-flow style.
- Existing boundaries between helpers, classes, modules, and tests.
- Tool-enforced formatting versus judgment-based style.

## Recommend

- Keep recommendations scoped to touched code.
- Prefer local clarity over generic clean-code slogans.
- Explain any intentional departure from local style.
